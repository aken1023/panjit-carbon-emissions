# 碳排管理 SaaS 系統架構文件

> 協助企業整合碳盤查、資料管理與揭露作業，建立一致、可信且可延續的碳管理體系，支援企業穩健推動淨零轉型。

---

## 一、系統定位與目標

### 核心主張
透過單一平台，企業可有效管理溫室氣體排放資料、完成盤查與報告作業，並將碳數據轉化為可支持決策與行動的管理基礎。

### 差異化定位
| 維度 | 國際平台（Watershed/Persefoni）| 國內平台（精誠/威煦）| **本平台** |
|------|-------------------------------|---------------------|-----------|
| 語言/UX | 英文為主 | 中文 | 繁體中文，現代化 SaaS UX |
| 台灣法規整合 | 薄弱 | 有 | 深度整合（碳費、TWSE、MOE）|
| 國際框架 | 完整 | 有限 | 完整（ISSB、CSRD、GHG Protocol）|
| Scope 3 供應鏈 | 強 | 弱 | 台灣供應鏈生態（台積電體系）|
| AI 自動化 | 強 | 弱 | AI 輔助盤查、異常偵測、報告草稿 |
| 定價 | 企業級高價 | 傳統授權 | 中小型上市公司可負擔的 SaaS |

## 二、目標客群

### 主要市場（台灣）

```
優先順序 1：半導體與電子製造供應鏈
  - 台積電供應商碳排要求（2025 起納入供應商評選）
  - 台灣上市公司 ~1,800 家（2025 年全面強制永續報告）
  - 台灣 MOE 強制申報：排放 >25,000 tCO2e 的製造業

優先順序 2：重工業（鋼鐵、石化、水泥）
  - 2025 年 1 月起繳納碳費（NT$300/tCO2e）
  - EU CBAM 影響台灣鋼鐵出口

優先順序 3：金融業
  - 金管會 TCFD 揭露要求
  - 融資排放（Financed Emissions）計算

優先順序 4：其他上市公司（零售、旅宿、醫療）
  - 2025 年 3 月 MOE 擴大強制申報範圍
```

### 次要市場（東南亞）
- 新加坡 SGX 碳揭露要求
- 日本 TCFD/ISSB 要求
- 韓國碳排放交易體系（K-ETS）

### M1：碳盤查管理
> **目標**：建立符合 ISO 14064-1 / GHG Protocol 的標準化盤查架構，讓企業每年可重複、可驗證地完成 GHG 盤查作業。

#### 1-1 盤查邊界設定
| 功能 | 說明 |
|------|------|
| 組織架構匯入 | 支援 Excel 批次匯入子公司、廠區、部門，建立樹狀組織圖 |
| 組織邊界方法 | 股權法（Equity Share）/ 財務控制權法 / 營運控制權法 三擇一 |
| 營運邊界設定 | 逐一確認 Scope 1、Scope 2（位置基礎 / 市場基礎）、Scope 3 各類別是否納入 |
| 邊界變更紀錄 | 年度邊界異動（新購廠、出售子公司）自動觸發歷史數據重算通知 |
| 重要性篩選 | 依排放量占比（如 >1%）自動標示需優先盤查之排放源 |

#### 1-2 排放源清單
```
Scope 1（直接排放）
  ├── 固定燃燒源：鍋爐、工業爐、發電機（燃煤、天然氣、柴油）
  ├── 移動燃燒源：公司自有車輛、堆高機、船舶
  ├── 製程排放：化學製程、冶煉、半導體製程氣體（F-gas、PFCs）
  └── 逸散排放：冷媒洩漏、廢水處理、儲槽呼吸閥

Scope 2（間接排放）
  ├── 外購電力（位置基礎：台電平均係數）
  ├── 外購電力（市場基礎：綠電憑證扣除）
  ├── 外購蒸汽 / 熱能 / 冷能
  └── 再生能源憑證（T-REC、I-REC 登錄管理）

Scope 3（價值鏈排放，15 類別）
  ├── Category 1：採購商品與服務
  ├── Category 2：資本財
  ├── Category 3：能源相關活動（不含 Scope 1/2）
  ├── Category 4：上游運輸與配送
  ├── Category 5：營運產生廢棄物
  ├── Category 6：員工差旅
  ├── Category 7：員工通勤
  ├── Category 11：售出產品使用階段
  └── ...（依企業性質啟用相關類別）
```

#### 1-3 排放係數資料庫
| 資料來源 | 涵蓋範圍 | 更新頻率 |
|---------|---------|---------|
| 台灣 EPA | 台電電力係數、燃料係數、冷媒 GWP 值 | 每年自動同步 |
| IPCC AR6 | 全球 GWP 100 年值、燃料排放係數 | 每次 IPCC 報告更新 |
| GHG Protocol | Scope 3 EEIO（經濟投入產出）係數 | 年度更新 |
| DEFRA (英國) | 差旅、運輸業係數（國際差旅常用）| 每季更新 |
| 自訂係數 | 企業自有實測數據（第一手資料，優先採用）| 手動維護 |

**係數版本管理**：歷史盤查期間鎖定當年係數版本，新盤查自動套用最新版，並提示差異影響。

#### 1-4 盤查任務管理
```
盤查期間（如 2025 年度）
  ├── 任務建立：系統依排放源清單自動產生填報任務
  ├── 任務指派：指定各排放源的資料填報負責人（含截止日）
  ├── 進度追蹤：儀表板顯示各單位填報完成率（% 完成）
  ├── 提醒通知：截止前 14 天 / 7 天 / 1 天自動發送 Email 提醒
  └── 補件要求：主管可退回任務並附上補件說明
```

#### 1-5 稽核軌跡（Audit Trail）
每筆排放量紀錄完整保存：
- 原始活動數據（如：用電度數 12,500 kWh）
- 套用排放係數（如：0.494 kgCO2e/kWh，2024 版）
- 計算結果（12,500 × 0.494 = 6,175 kgCO2e）
- 填報人、填報時間、佐證文件連結
- 修改紀錄（誰改了什麼、改前改後對比）

### M2：資料採集
> **目標**：降低資料蒐集的人工作業負擔，提供多元輸入管道，確保資料完整性與可追溯性。

#### 2-1 手動填報表單
- 依排放源類型提供對應欄位（如：燃料種類、用量、單位）
- 動態欄位驗證（負數、超合理範圍自動警示）
- 支援月份分拆輸入（年度數據可按月填報後自動加總）
- 佐證文件必填控制（可設定哪些排放源需強制附件）

#### 2-2 Excel / CSV 批量上傳
```
上傳流程：
1. 下載系統標準範本（依排放源類型）
2. 填寫後上傳，系統解析並顯示預覽
3. 欄位映射確認（若使用自訂格式）
4. 批次驗證：列出所有錯誤行 + 錯誤原因
5. 確認後批次匯入，產生匯入紀錄
```

#### 2-3 OCR 文件自動解析
| 文件類型 | 解析欄位 | 準確率目標 |
|---------|---------|---------|
| 台電電費通知單 | 用電度數、計費期間、電號 | >95% |
| 柴油/天然氣採購發票 | 數量、單位、日期、廠商 | >90% |
| 冷媒採購/加充紀錄 | 冷媒型號、重量（kg）| >90% |
| 車輛加油收據 | 油種、公升數、日期 | >88% |

**操作流程**：上傳 PDF/圖片 → AI 解析 → 人工確認關鍵欄位 → 確認後存入活動數據

#### 2-4 ERP 連接器
| ERP 系統 | 整合方式 | 自動抓取資料 |
|---------|---------|------------|
| SAP S/4HANA | API / RFC | 能源採購訂單、生產數量、差旅費用 |
| 鼎新 ERP | API | 採購金額（Scope 3 spend-based 估算用）|
| Oracle Fusion | REST API | 電力費用、燃料採購 |
| 自訂 ERP | CSV 定時排程 | 設定欄位映射後定期匯入 |

#### 2-5 IoT / 智慧電錶整合
- 支援 MQTT / REST API 即時或定時拉取能耗數據
- 可設定廠區 → 電錶 → 排放源的對應關係
- 異常讀值自動偵測（如：週末用電量異常高）
- 歷史數據補拋機制（設備離線後重連自動補傳）

#### 2-6 資料品質管理
```
三層品質檢查：
① 格式驗證：數值類型、單位合法性、日期格式
② 邏輯驗證：月份加總 vs 年度總量、同廠區歷史趨勢對比
③ 合理性警示：超過歷史均值 ±30% 自動標記「待確認」，需填報人說明原因
```

### M3：揭露報告
> **目標**：將盤查數據自動轉換為各監管機構要求的申報格式，消除手動整理報告的時間與錯誤風險。

#### 3-1 支援報告框架一覽

| 框架 | 用途 | 輸出格式 | 截止日（台灣）|
|------|------|---------|-------------|
| **環境部 GHG 申報** | MOE 法定申報 | 系統介接上傳 / XML | 每年 4 月 30 日 |
| **金管會永續報告書** | TWSE/TPEx 上市公司 | Word / PDF | 每年 8 月 31 日 |
| **ISSB / IFRS S2** | 氣候相關財務揭露（2026起）| Word / PDF | 配合年報 |
| **GHG Protocol** | 企業標準盤查 | Excel / PDF | 依公司政策 |
| **ISO 14064-1** | 查核用工作底稿 | Excel + PDF | 依查核合約 |
| **TCFD** | 四大支柱揭露 | PDF / Word | 配合年報 |
| **GRI 305** | 永續報告書 Emissions 章節 | Word | 依公司政策 |
| **CDP 問卷** | Carbon Disclosure Project | CDP 線上系統格式 | 每年 7 月 |

#### 3-2 報告產生器
```
使用者操作流程：
1. 選擇盤查期間（如：2025 年度）
2. 選擇報告框架（如：金管會永續報告書）
3. 確認章節範圍（可勾選納入/排除特定章節）
4. 選擇語言（繁體中文 / English）
5. AI 自動填入數據表格 + 生成敘述性文字草稿
6. 人工審閱修改
7. 匯出 PDF / Word / Excel
```

#### 3-3 第三方查核工作流
```
查核流程管理：
企業設定查核機構（如 BSI、SGS、Bureau Veritas）
  → 系統產生查核工作底稿包（含原始數據 + 計算過程 + 佐證文件清單）
  → 查核機構登入專屬查核空間（唯讀 + 意見填寫）
  → 查核意見逐條回應追蹤（Open / In Review / Resolved）
  → 查核聲明書上傳，觸發報告最終鎖定
  → 鎖定後任何修改需走「更正申請」流程
```

#### 3-4 歷年數據比較
- 自動生成多年度趨勢表（Scope 1/2/3 各別 + 合計）
- 碳密度指標（tCO2e / 營收、tCO2e / 產品產量）年度對比
- 與行業基準比較（依 GICS 產業分類提供參考值）

### M4：減碳管理
> **目標**：讓碳數據不只是申報工具，而是驅動企業實際減碳行動的決策基礎。

#### 4-1 基準年與目標設定
| 功能 | 說明 |
|------|------|
| 基準年鎖定 | 設定基準年（通常為進行盤查的第一年），鎖定後不可修改 |
| 重新計算基準年 | 若組織邊界重大變更（如：購併），系統提示是否需重算基準年 |
| 減量目標設定 | 絕對量減少目標（如：2030 年較 2020 年減少 42%）或強度目標 |
| SBTi 對齊工具 | 依 SBTi 方法論計算 1.5°C / Well-below 2°C 目標值，生成 SBTi 提交所需文件 |
| 年度里程碑 | 設定每年減量里程碑，自動計算是否達標 |

#### 4-2 情境模擬（Scenario Planning）
```
模擬情境範例：
  情境 A（基準）：不採取額外行動
  情境 B：2026 年導入太陽能屋頂（預估減少 XX tCO2e）
  情境 C：2027 年切換綠電合約（100% 再生能源）
  情境 D：B + C 組合

系統輸出：
  → 各情境下 2025–2030 年排放量預測折線圖
  → 距目標的缺口（Gap Analysis）
  → 各行動的估計成本 vs 減碳效益（NT$/tCO2e）
```

#### 4-3 減碳專案管理
每個減碳行動作為一個「專案」登錄管理：

| 欄位 | 說明 |
|------|------|
| 專案名稱 | 如：廠區屋頂太陽能安裝 |
| 專案類型 | 能效提升 / 燃料替代 / 製程改善 / 再生能源 / 碳抵銷 |
| 預計執行期間 | 開始 ~ 完成日 |
| 預計減碳量 | tCO2e / 年（估算值）|
| 實際減碳量 | 系統自動從盤查數據計算（執行後）|
| 投資金額 | NT$ |
| 邊際減碳成本 | 自動計算 NT$/tCO2e |
| 狀態 | 規劃中 / 執行中 / 完成 / 取消 |
| 負責人 & 里程碑 | 任務分派與進度追蹤 |

#### 4-4 碳權 / 碳抵銷管理
```
支援碳權類型：
  ├── 國內：台灣碳權交易所（TCXC）碳權
  ├── 國際：Gold Standard、VCS（Verra）、CDM 減量憑證
  └── 再生能源：T-REC（台灣）、I-REC（國際）

管理功能：
  ├── 碳權憑證登錄（序號、發行機構、有效期）
  ├── 使用紀錄（抵銷哪個年度的哪個 Scope）
  ├── 抵銷後淨排放量自動重算
  └── 碳權到期提醒
```

#### 4-5 減碳進度儀表板
- 當年排放量 vs 年度目標 → 差距（tCO2e）與達成率（%）
- 減碳專案執行狀態彙整（甘特圖 + 進度）
- 累計減碳效益 vs 基準年
- 淨零路徑圖（Glide Path）視覺化

### M5：供應鏈（Scope 3）管理
> **目標**：解決企業最難計算的 Scope 3 排放問題，透過系統化供應商資料收集取代估算，提升數據品質。

#### 5-1 Scope 3 類別啟用設定
依企業性質（製造業、金融業、零售業）提供建議清單，企業勾選適用類別：

| 類別 | 名稱 | 常用計算方式 |
|------|------|------------|
| Cat.1 | 採購商品與服務 | 供應商自報 / Spend-based |
| Cat.2 | 資本財 | 供應商自報 / Spend-based |
| Cat.3 | 燃料與能源相關活動 | 活動數據法 |
| Cat.4 | 上游運輸 | 重量 × 距離 × 運輸模式係數 |
| Cat.5 | 廢棄物 | 廢棄物類型 × 處理方式係數 |
| Cat.6 | 員工差旅 | 航班里程 / 住宿間夜 × 係數 |
| Cat.7 | 員工通勤 | 通勤調查問卷 + 交通模式係數 |
| Cat.11 | 售出產品使用 | 產品能耗 × 預期壽命 × 使用量 |
| Cat.15 | 投資（金融業）| PCAF 方法論（融資排放）|

#### 5-2 供應商入口網站

**供應商收到邀請郵件流程：**
```
企業設定採購商品清單 + 供應商聯絡人
  → 系統自動寄送邀請信（含專屬連結，無需註冊帳號）
  → 供應商點擊連結進入填報頁面
  → 填寫以下資訊：
      ① 年度總排放量（Scope 1+2）或產品碳足跡
      ② 計算方法（自行計算 / 已取得查核聲明）
      ③ 查核聲明書上傳（如有）
      ④ 使用的排放係數來源
  → 提交後企業端自動收到通知，資料納入 Cat.1 計算
```

**供應商管理功能：**
| 功能 | 說明 |
|------|------|
| 填報狀態追蹤 | 已邀請 / 已填報 / 已逾期 / 資料有疑問 |
| 催填提醒 | 系統自動在截止前發送第 2、3 次提醒 |
| 資料品質評分 | 依資料來源（查核聲明 > 自算 > 估算）自動評分 |
| 歷年比較 | 供應商連續兩年數據差異超過 20% 自動標記 |
| 供應商基準庫 | 建立同產業供應商排放強度基準，未填報者自動套用 |

#### 5-3 資料品質層級（DQS）

```
Level 1（最高）：供應商提供有查核聲明的排放數據
Level 2：供應商自行計算並提交（無查核）
Level 3：使用行業平均排放強度（次級數據）
Level 4：Spend-based 估算（採購金額 × EEIO 係數）
```

報告中自動標示各類別的整體資料品質分布，以符合 GHG Protocol 揭露要求。

#### 5-4 供應商熱點分析（Hotspot）
- 依 Cat.1 採購金額排名前 20 大供應商的排放占比
- 識別「高採購金額 × 高排放強度」的優先合作對象
- 生成供應商減碳合作邀請簡報（一鍵匯出）

### M6：碳費管理（台灣特有）
> **目標**：協助台灣碳費納管企業（>25,000 tCO2e）精確計算應繳碳費，規劃自主減量計畫以適用優惠費率，降低碳費財務衝擊。

#### 6-1 碳費應繳金額計算
```
計算邏輯：
  應繳排放量 = 申報排放量 - 核配免費額度
  應繳碳費 = 應繳排放量 × 費率

費率情境（台灣環境部，2025）：
  一般費率：NT$300 / tCO2e
  自主減量計畫優惠費率 A：NT$50 / tCO2e（達標條件：2030 年減 22%）
  自主減量計畫優惠費率 B：NT$100 / tCO2e（達標條件：2030 年減 10%）
```

| 儀表板顯示 | 說明 |
|-----------|------|
| 本年度預估碳費 | 依目前盤查進度即時試算 |
| 費率情境比較 | 一般費率 vs 優惠費率 A vs 優惠費率 B 之碳費差異 |
| 3 年碳費走勢 | 依費率調升計畫預估未來 3 年財務影響 |
| 減碳 ROI 計算 | 投入減碳行動成本 vs 節省碳費金額比較 |

#### 6-2 自主減量計畫（SRP）支援
- 依環境部 SRP 申請規定，自動產生計畫書所需數據章節
- 減量目標可行性評估（依現有減碳專案預估達標機率）
- SRP 申請進度管理（準備中 / 已提交 / 審核中 / 核准 / 退件）
- 核准後費率自動切換為優惠費率計算

#### 6-3 申報時程管理
```
關鍵時程提醒（以台灣現行規定）：
  ├── 每年 4 月 30 日 → GHG 年度申報截止
  ├── 每年 10 月 31 日 → 第三方查核完成截止
  ├── 碳費繳納截止日 → 系統設定提醒
  └── SRP 申請截止日 → 依環境部公告自動更新
```

#### 6-4 碳費繳費紀錄
- 每年繳費金額 / 費率 / 納管排放量歷史紀錄
- 繳費憑證上傳保存
- 多年度碳費趨勢圖（用於 TCFD 財務影響揭露）

### M7：AI 助理
> **目標**：透過 AI 降低碳管理的專業門檻，自動化高重複性作業，讓非專業人員也能完成高品質的碳管理工作。

#### 7-1 智慧排放係數匹配
```
使用場景：企業上傳採購清單（物料名稱、數量、單位）
AI 處理流程：
  1. 解析物料名稱（如：「高密度聚乙烯（HDPE）顆粒」）
  2. 對應至係數資料庫的最佳匹配（如：Plastics manufacturing, HDPE）
  3. 列出匹配結果 + 信心度分數（High / Medium / Low）
  4. Low 信心度項目提示人工確認
  5. 確認後批量產生 Cat.1 排放量計算
```

#### 7-2 異常偵測與根因分析
| 觸發條件 | AI 回應 |
|---------|---------|
| 月度電力排放量 > 歷史均值 +30% | 標記異常，列出可能原因（產量增加？設備異常？填報錯誤？）|
| 某排放源連續 3 個月無數據 | 提示可能遺漏填報 |
| 年度總排放量 vs 去年差異 >20% | 自動分析差異來源（哪個 Scope / 哪個廠區貢獻最大）|
| 供應商填報數值 vs 行業基準差異 >2 倍 | 標記供應商數據需人工複核 |

#### 7-3 報告文字草稿生成
```
功能：依盤查數據自動生成各框架的敘述性文字

範例：輸入盤查數據後，AI 產出 TCFD 報告草稿：

「本公司 2025 年度溫室氣體排放總量為 X,XXX tCO2e，較基準年（2020年）
減少 X%。其中 Scope 1 直接排放為 XXX tCO2e，主要來源為廠區製程用天然氣；
Scope 2 間接排放為 XXX tCO2e，本年度已購入 XX 張台灣再生能源憑證（T-REC）...」

使用者可直接編輯草稿或指示 AI 調整語氣 / 補充說明。
```

#### 7-4 法規解讀助手
- 輸入法規名稱或條文，AI 解釋對企業的具體義務
- 定期推播法規更新摘要（如：環境部費率調整、TWSE 新要求）
- Q&A 模式：「我們是製造業，要申報哪些 Scope 3 類別？」

#### 7-5 減碳建議引擎
```
基於企業的排放熱點分析，AI 提供優先減碳行動建議：

範例輸出：
  高優先（高減量 / 低成本）：
    1. 申請綠電合約（預估減少 Scope 2 XX%，費用 NT$XXX/月）
    2. 汰換老舊壓縮機（預估減少 Scope 1 X%，投資回收期 3 年）

  中優先：
    3. 建立員工通勤補貼（大眾運輸），改善 Cat.7
    4. 要求前 10 大供應商填報排放數據

（每項建議附上預估減碳量、投資金額、補助資訊連結）
```

## 六、核心業務流程

### 年度碳盤查流程
```
1. 設定盤查邊界
   → 組織架構匯入 → 邊界類型選擇（股權法/控制權法）

2. 排放源識別
   → 系統建議清單 → 確認適用排放源 → 指派資料填報負責人

3. 活動數據採集
   → 各單位填報 / ERP 自動匯入 / OCR 解析 → 上傳佐證文件

4. 排放量計算
   → 系統自動匹配排放係數 → 計算 CO2e → 品質審查

5. 內部審查
   → 碳管理主管確認 → 異常值處理 → 簽核

6. 第三方查核
   → 匯出查核工作底稿 → 查核機構線上協作 → 查核聲明書上傳

7. 揭露申報
   → 環境部線上申報格式匯出 → TWSE 永續報告書產出 → CDP 問卷填寫
```

### 供應商 Scope 3 資料收集流程
```
企業設定 Category 1 採購商品清單
  → 系統寄送邀請信給供應商
  → 供應商透過入口網站填報 GHG 資料 + 上傳文件
  → 系統驗證資料品質
  → 未回應者自動套用 Spend-based 估算
  → 彙整至企業 Scope 3 Category 1 排放量
```
